 # UI/UX 设计方针

以下以备忘录的形式，分享 VOICEVOX 的 UX 和 UI 设计理念。

## 基本方针

- 最重要目标是“确保用户能够完成他们想要创作的内容”。
- 基于这一理念，我们在设计时注重以下几点：
  - 让用户感到“想使用”的吸引力。
  - 确保工作流程清晰易懂。
  - 尽量减少操作时的思考负担。
- 同时，我们也重视“帮助用户创作更优质的内容”。
- 基于这一理念，我们还注重：
  - 让用户能够轻松提升语音质量。
  - 打造具有创造性的软件。

以下介绍这些理念如何落实到具体实现中。

- **让用户感到“想使用”的吸引力**
  - 为了让更多人能够接触到软件，我们提供免费分发，并尽可能提升品质。
  - 为了避免让人觉得“过时”，我们追求较新颖的外观设计。
  - 为了不给未来用户留下负面印象，我们在 SNS 上尽量避免发布负面情绪，并积极解答用户的疑问。
- **确保工作流程清晰易懂**
  - 注重从文本输入到语音导出的第一视角体验。
  - 对可有可无的功能，尽量设计为不显眼的 UI。
- **尽量减少操作时的思考负担**
  - 注重简洁的 UI 设计。
  - 避免使用复杂术语，力求简洁的表达。
  - 确保无需了解快捷键也能轻松操作的 UI。
- **让用户能够轻松提升语音质量**
  - 设计第一视角时，确保用户能直观了解可以调整语音。

### 使命与软件方针的对应

- 我们认为实现[使命](./ミッション・バリュー・ビジョン.md#ミッション)最重要的是 UGC（用户生成内容）。
  - 用户创作的内容能够扩大社区影响是我们的理念。
- 基于此，软件的目标是让用户创作出更多更优质的内容。

## 设计决策流程

目前仍在不断尝试改进，但大致流程如下：

1. **通过 Issue 等提出设计外观**  
   - 此步骤旨在尽早发现遗漏或需要考虑的问题。  
   - 可以是文本、手绘图或样例实现，形式不限。
2. **实际实现并创建 Pull Request**  
   - UI/UX 设计很多细节需通过实际操作验证，因此需要实现。  
   - 即使代码量少，设计变更的内容可能较多，建议将差异控制在正负 80 行左右，以确保合并顺利。
3. **识别问题点**  
4. **合并**  
5. **根据整体设计需要，由维护者进行微调**

## 详细设计指南

- 取消按钮置于左侧。
- 对话框与提示（Toast）的使用区分：
  - 由用户操作引发且错误或频率较低的反馈使用对话框（`SHOW_DIALOG_` 系列）。  
    - 示例：［需要重启引擎］引擎已停止，无法生成语音。是否重启引擎？［不重启/重启］
  - 其他情况使用提示通知（`SHOW_NOTIFY_` 系列）。  
    - 示例：已导出语音。  
    - 示例：引擎异常终止，需重启引擎。
- 关于对话框：
  - **warning 与 confirm 的区分**  
    - 若继续操作更可能被期待，使用 confirm。  
    - 若取消操作更可能被期待，使用 warning。
  - **对话框文案**  
    - 标题使用疑问句，如“是否～？”  
    - 正文力求简洁，避免使用“是否可以？”等表达。
  - **按钮颜色**  
    - 肯定的、用户可能更倾向选择的按钮使用 Primary 色。  
    - 破坏性的、恢复状态需耗费精力的操作按钮使用 Warning 色。

## 文案相关

### VOICEVOX 专用 UI 名称

标有 🚷 的名称暂未面向用户（不用于用户指引）。

具体文件对应关系请参阅 [UI 名称与 Vue 文件名的对应速查表](UI名称とVueファイル名の対応早見表.md)。

- **菜单** ･･･ 最上方的工具栏，包含文件、设置等项目。
  - **固定按钮 🚷** ･･･ 关闭按钮旁的最前面固定功能按钮。
- **工具栏** ･･･ 菜单下方的工具栏，可自定义。
- **角色显示栏 🚷** ･･･ 屏幕左侧的面板。
- **台词栏 🚷** ･･･ 包含文本框等的面板。
  - **轨道 🚷** ･･･ 包含单行文本的区域，面向用户时有时称为“文本栏”。
    - **行号** ･･･ 轨道的编号，若面向用户，可能称为“轨道编号”。
    - **角色图标** ･･･ 可更改角色或风格的按钮。
    - **文本栏** ･･･ 输入用于语音合成的文本区域。
  - **文本添加按钮** ･･･ 添加轨道的按钮。
- **参数调整栏 🚷** ･･･ 屏幕右侧可调整参数的面板。
- **详细调整栏 🚷** ･･･ 屏幕下方可调整语调等的面板。
  - **项目** ･･･ 详细调整栏中语调或重音的选择按钮。
  - **播放按钮** ･･･ 可播放语音的按钮。
  - **重音区间** ･･･ 显示一个重音句的区域（期待更合适的名称）。
  - **文字区间 🚷** ･･･ 重音区间的单文字显示区域，面向用户时有时称为“按文字”。
  - **朗读** ･･･ 显示假名的区域。

#### 其他 UI 部件

- **栏** ･･･ 视觉上分隔的区域。
- **参数直接输入 🚷** ･･･ 点击参数可直接输入的区域，与普通文本显示无异，暂无专用名称。
- **滑块** ･･･ 用于调整重音、语速等的控件。
- **选中轨道/重音区间 🚷** ･･･ 当前激活的轨道或重音区间。

### 其他概念名称

- 面向用户时，使用“文件夹”而非“目录”。
- 多引擎相关：
  - 符合 VOICEVOX API 的引擎称为“引擎”。  
    - 若无法区分是 VOICEVOX 引擎还是兼容引擎，称为“VOICEVOX 兼容引擎”。
  - VVPP 文件直接称为“VVPP 文件”，如需解释，可称“VOICEVOX 兼容引擎包”。
- 使用与 VOICEVOX 编辑器相同 UI 的软件统称为“VOICEVOX 系软件”。  
  - 正式名称为“VOICEVOX 系 UI 软件”，因较长，基本不用。

### 文体

- 字母使用半角。
- 括号使用全角。  
  - 但若括号内全为半角字符，则括号也使用半角。
- 使用“です/ます”敬体。
- 项目名称以名词结尾（体言止）。
- 按钮内容为“～する”形式。

### 提示（Tooltip）

- 文末加句号。
- 文末有括号时，句号置于括号外。  
  - 示例：如此操作（但需注意）。  
- 单选框（radio）：使用“可选择。”  
- 下拉框（select）：使用“可更改。”  
- 按钮（button）：使用“可～。”  
- 复选框（checkbox）：  
  - ○○ 功能：使用“启用 ○○ 功能。将会～。”  
  - 默认 OFF 时：使用“开启时，将会～/不会～。”  
  - 默认 ON 时：使用“关闭时，不会～/将会～。”
